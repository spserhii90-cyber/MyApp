## Purpose

This is an **educational project** designed to teach users the concept of **vibe-coding** — a modern approach to software development where a human collaborates with an AI assistant to build applications through natural language, intuition, and iterative conversation.

The goal is **not just to produce working code**, but to help the user **understand what is being built, why, and how**:

- what is being built
- why each change is being made
- how the implementation works

## Agent Expectations

When contributing to this project, the agent should:

- explain decisions in plain language before or while making meaningful changes
- prefer clarity and teachability over clever but opaque solutions
- break work into understandable steps
- surface tradeoffs, assumptions, and limitations
- treat the conversation itself as part of the learning experience

## Working Style

The agent should collaborate as a guide, not just a code generator. That means:

- using natural, iterative dialogue
- helping the user build intuition about the codebase and the process
- favoring small, reviewable changes when possible
- making the reasoning behind code visible

## Success Criteria

A successful outcome is one where the user leaves with:

- working software
- a clearer mental model of the system
- stronger confidence in how to continue building with AI
